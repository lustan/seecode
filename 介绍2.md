# 推荐个我自己一直在用的小工具：Seecode

后端写久了，每天都要跟 JSON 打交道。看接口返回、对比两版配置、临时存几段 SQL……这些事儿单拎出来都不大，但加一块儿挺烦的。

以前我的流程是这样：

- 看 JSON → 打开在线格式化网站，担心敏感数据被传出去
- 对比两段文本 → 又打开一个 diff 网站
- 临时记代码片段 → 新建个 VS Code 窗口，写完关掉

后来索性自己折腾了个 Chrome 插件，叫 **Seecode**，把这几件事塞一块儿了。用了几个月觉得还挺顺手的，分享一下。

---

## 它能干啥

说白了就三件事：**格式化 JSON、记代码片段、对比两段文本**。

但都是浏览器里直接搞定，不用切窗口、不用上传到外网。

## 几个我用得最多的点

### 1. JSON 格式化能"修"那种半坏的

接口返回的 JSON 经常带尾逗号、或者键没加引号，正经格式化工具直接报错。Seecode 的格式化是带容错的，多数情况能给你修好再格式化。`Ctrl+J` 一下，省事。

压成单行也是一键（`Ctrl+K`），准备贴回代码或者塞配置文件里很方便。

### 2. 笔记当便签用，还能拖出来悬浮

支持 JSON、JS、Java、Python、SQL、Markdown，粘贴进去自动识别语言并高亮。

最爽的是任何一条笔记都能 **拖出来变成独立的悬浮小窗**，置顶在屏幕角落。我经常拿来贴接口文档、临时 SQL，写代码时瞄一眼就行，不用 alt-tab 切来切去。

### 3. 文件对比（这次重点更新）

之前对比两段文本我都是用网页版 diff 工具。现在直接在插件里就能做，而且做得比那些网站讲究：

- **两边都能编辑**——不是只读 diff，边改边看差异变化
- **滚动联动**——左边滚到哪儿，右边自动跟上，不会错位
- **F7 / Shift+F7** 在差异块之间来回跳，键盘党会喜欢
- 顶上有个统计：删了几行、改了几行、加了几行
- 一键 Swap 翻转左右
- 每边都有独立的 Format / Copy / Clear

红绿黄三色块标差异，跟 IDEA 那个 Compare Files 几乎一模一样的体感。

平时我用它做两件事：

- 对比两个版本的接口返回
- 提 PR 前先自己看一遍 diff，心里有数

## 一些细节

- 编辑器底子是 CodeMirror 6，几千行的 JSON 丢进去不卡。
- 主题有 IDEA Dark 和 GitHub Light 两套，看心情切。
- 搜索替换支持正则和大小写敏感（`Ctrl+F` / `Ctrl+R`）。
- 数据都存本地，不联网、不上传。

---

## 安装

目前还没上 Chrome 商店，开发者模式手动装一下：

1. 去 GitHub 下载或 clone 代码
2. 打开 `chrome://extensions/`
3. 右上角打开"开发者模式"
4. 点"加载已解压的扩展程序"，选项目目录

完事，工具栏就有图标了。
